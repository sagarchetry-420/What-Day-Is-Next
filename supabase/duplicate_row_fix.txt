ISSUE: ON CONFLICT DO UPDATE command cannot affect row a second time
======================================================================

CAUSE:
This error occurs when the same unique key combination (name, holiday_date,
country_code, region) appears multiple times within a single upsert batch.
PostgreSQL does not allow updating the same row twice in one statement.

FIX APPLIED (Code-side - No Supabase changes needed):
------------------------------------------------------
The fix deduplicates rows before upserting by keeping only the last occurrence
of each unique key combination.

Files modified:
1. server/holidaySync.js - Added deduplicateRows() function
2. server/index.js - Added inline deduplication before batch upsert

The deduplication uses a Map with the conflict key as the key:
  key = `${name}|${holiday_date}|${country_code}|${region}`

This ensures each batch only contains unique rows, preventing the PostgreSQL error.

NO DATABASE MIGRATION REQUIRED.
